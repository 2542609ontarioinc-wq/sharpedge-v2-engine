src/grading/subscriber_thresholds.py
  Single source of truth for qualification thresholds. Both the
  Python grading chain and the web filter read from here so they
  can never drift apart.
    EDGE_MIN  = 3.0    (min model edge % for sharp/prop picks)
    PROB_MIN  = 65.0   (min win-probability % for all pick types)
    BOTD_EDGE = 5.0    (Bet of the Day: edge must be >= this)
    BOTD_PROB = 70.0   (Bet of the Day: win prob must be >= this)
    EDGE_MAX  = 15.0   (upper sanity cap, mirrors web isRealValue check)

src/grading/build_mlb_pick_detail.py
  After grading, enriches every graded game pick row with two new
  boolean columns: subscriber_qualified and bet_of_day.
  Sharp picks qualify when: edge_flag == "REAL" AND edge >= EDGE_MIN
  AND prob >= PROB_MIN. Safe-zone picks qualify on prob >= PROB_MIN
  alone (no model edge). Bet of the Day requires edge >= BOTD_EDGE
  AND prob >= BOTD_PROB.

src/grading/build_mlb_prop_detail.py
  Same subscriber flag logic for player prop picks. Pick side
  (Over/Under) is used to flip calibrated_over_prob to the correct
  win-probability for the chosen side before applying PROB_MIN.

src/analytics/build_mlb_subscriber_analytics.py
  Reads subscriber_qualified = true rows from both detail tables,
  aggregates across two segments ("all" and "bet_of_day"), and
  upserts one row per segment into mlb_subscriber_results.
  Reports: N, W-L, win%, ROI, avg edge, avg win-prob, avg CLV,
  CLV beat rate. CLV is framed as the primary signal; win%/ROI
  are secondary (favourites bias caveat).

web/src/components/MLBSubscriberView.tsx

web/src/lib/data.ts → getMLBSubscriberResults()
