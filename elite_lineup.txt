CHANGE 2 — Lineup-confirmed badge + auto-expand props
─────────────────────────────────────────────────────────────────────────────
BACKGROUND
  Player props are generated only after a game's batting lineup is confirmed
  (~90 min pre-game via the hourly job). Batter props (h_r_rbi market,
  playerType === "batter") are generated when the batting order is submitted.
  Pitcher-only props are generated earlier and do NOT indicate lineup confirmation.

DETECTION
  In MLBSubscriberView (before rendering):

    const lineupConfirmedGameIds = new Set(
      playerProps
        .filter((p) => p.playerType === "batter")
        .map((p) => p.gameId)
    );

  A game is lineup-confirmed if it has at least one batter prop in the raw
  playerProps array (before qualifying-threshold filtering). This is correct
  because the presence of batter props in the DB is the proxy for lineup lock.

  lineupConfirmedGameIds is passed to each GameCard as:
    lineupConfirmed={lineupConfirmedGameIds.has(g.gameId)}

GAME CARD CHANGES (GameCard component)
  Props section is now COLLAPSIBLE, separate from non-prop picks:

  - nonPropPicks (sharp / balanced / banker):
      Always rendered in the main picks area. Unchanged.

  - propPicks (props):
      Rendered inside a collapsible section with a toggle button.
      Initial open state:
        const [propsOpen, setPropsOpen] = useState(lineupConfirmed && hasPropPicks);
      — auto-expanded when lineup is confirmed AND qualifying props exist.
      — collapsed by default when lineup is not confirmed.

  Game header (below team names / game time):
    Lineup confirmed  → "✓ Lineup confirmed" in text-elite (green)
    Lineup pending    → "Lineup pending" in text-muted/45 (shown only if
                        there are visible prop picks — i.e. pitcher-only props)
    No props at all   → no lineup status shown

  Props toggle button shows:
    "⚾ Player Props  [N]  ✓ Lineup confirmed"  when confirmed
    "⚾ Player Props  [N]  Lineup pending"       when not confirmed
    "▲" / "▼" chevron on the right

  The 2-props-per-game cap (MAX_PROPS_PER_GAME = 2) is applied before the
  split, so the capped count is what appears in the toggle button and what
  renders in the collapsed/expanded section.

STATES MATRIX
  lineupConfirmed = true  + propPicks.length > 0  → props section visible, auto-expanded
  lineupConfirmed = true  + propPicks.length = 0  → "✓ Lineup confirmed" badge only, no toggle
  lineupConfirmed = false + propPicks.length > 0  → "Lineup pending" badge + toggle, collapsed
  lineupConfirmed = false + propPicks.length = 0  → no lineup badge, no toggle
