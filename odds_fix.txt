ROOT CAUSE (from odds_clv_diagnosis.txt)
----------------------------------------------------------------------
build_mlb_final_picks.py re-processed ALL games in mlb_run_predictions
every daily run, including settled games. sync_mlb_odds.py purges
mlb_odds at the start of each run and only refetches upcoming games,
so settled games had no odds. The pick builder would find no odds,
set edge_flag='no-odds' and model_edge=None, and upsert that over the
original REAL prediction — destroying edge_flag, model_edge, odds_decimal,
market_implied_probability (opening novig), and mlb_safe_zone.balanced_novig_pct.

grade_mlb_picks.py reads edge_flag and model_edge directly from
mlb_final_predictions and copies them into mlb_pick_grades, so 100%
of graded picks inherited no-odds/null from the corrupted predictions.
build_mlb_clv.py then skipped all picks at the no-odds gate, producing
zero CLV rows.


FILE CHANGED

src/models/build_mlb_final_picks.py  (+86 lines, -7 lines)


WHAT WAS ADDED
----------------------------------------------------------------------

1. _SETTLED_STATUSES constant (module level)
   Set of lowercase status strings that mean a game is done:
     ft, aot, f, final, game finished, finished,
     postponed, cancelled, canceled, suspended, post

2. _is_settled(game) helper
   Returns True if game['status'] or game['period'] (lowercased) is
   in _SETTLED_STATUSES. Checks both fields to match the pattern used
   by grade_mlb_picks.py.

3. Layer 1 — skip settled games (primary fix), added to main()
   After fetching mlb_run_predictions, queries the games table for the
   status/period of every game_id in the prediction list and builds a
   settled_ids set. Any game in that set is skipped with 'continue'
   before candidates are built, before odds are looked up, and before
   any upsert fires. The stored row is completely untouched.

4. Layer 2 — non-destructive upsert (backstop), added to main()
   Before the main() loop, pre-loads existing edge_flag from
   mlb_final_predictions and balanced_edge_flag from mlb_safe_zone
   into dicts keyed by game_id. In Pass 3, before each upsert:
     - If existing edge_flag in ('REAL','suspect') AND new edge_flag
       == 'no-odds': block the mlb_final_predictions write, log it,
       increment skipped_downgrade counter.
     - Same guard for mlb_safe_zone using balanced_edge_flag.
   This catches the window between a game ending and the games table
   status being updated (i.e. the game slips through Layer 1 but
   already has real odds stored from an earlier run that day).

   Layer 2 logic covers all five cases correctly:
     REAL existing    → no-odds new  : BLOCKED   (protects real edge)
     REAL existing    → REAL new     : ALLOWED   (odds refresh ok)
     no-odds existing → no-odds new  : ALLOWED   (no data to protect)
     suspect existing → no-odds new  : BLOCKED   (protects opening novig)
     no existing row  → no-odds new  : ALLOWED   (first write, nothing to lose)

5. Updated summary line
   Now prints: N written | N safe zones | N settled skipped (Layer 1)
               | N downgrade blocked (Layer 2) | N no candidates

- grade_mlb_picks.py: unchanged. It reads from mlb_final_predictions,
  which will now have preserved REAL values for settled games.

- mlb_safe_zone: the balanced_novig_pct / banker_novig_pct fields that

- No soccer files touched.
- No model math, grading formulas, or pick selection logic changed.
