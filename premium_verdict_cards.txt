PER-PICK FIELD SET (component: PickFieldSet)
--------------------------------------------
Every pick — game card panel AND BotD hero — renders all 9 fields:

REAL fields (from actual model/data — never invented):
  1. Model Probability   pick.winProb (calibratedConf/calibratedConfidence)
  2. Book Implied Prob   computed via bookImplied(decimal, american)
                         formula: 1/decimal*100 or american convention
                         shows "no-odds" when both inputs are null
  3. Edge                pick.edge (modelEdge from mlb_final_predictions)
                         shown as "+X.X%" in green (#12f38b)
  4. Grade               derived from edge via edgeGradeLabel():
                           Elite  ≥ 10%
                           Strong ≥  8%
                           Play   ≥  5%
                           Lean   ≥  3%
  5. Odds                fmtOdds(american, decimal) — american preferred
  6. CLV                 always "—" for today's ungraded picks (honest pending
                         state; CLV is added post-grade, never fabricated)

BUILDING fields (shown honestly until n ≥ 30 graded samples):
  7. Hit Rate (Historical)  buildingValue(n, winRate%) → "Building (N/30)"
                            until n ≥ 30; then shows real win% for that market
  8. Comparables            always "Building — needs backtest data"
                            (requires full backtest pipeline, not yet built)
  9. Expected ROI           buildingValue(n, roi%) → "Building (N/30)"
                            until n ≥ 30; then shows real ROI% for that market

PickFieldSet props:
  pick        SubPick
  marketStats Map<string, MarketGradedStats>  (from computeMarketGradedStats)
  hero        boolean  (true = 2-col grid + large numbers for BotD)


BUILDING-FIELD GATING LOGIC
----------------------------
Type:
  MarketGradedStats { n: number; winRate: number|null; roi: number|null }

Function: computeMarketGradedStats(gradedPicks: MLBPickDetail[])
  - Iterates gradedPicks (passed as prop to MLBSubscriberView)
  - Filters with isQualified() — same gate as subscriber display filter:
      edgeFlag === "REAL"
      AND (game market → modelEdge ≥ 3% AND calibratedConf ≥ 65%)
      OR  (safe market → market in {safe_balanced, safe_banker})
  - Counts only grade === "WIN" | "LOSS" (voids excluded)
  - Groups by market (moneyline / totals / run_line / prop / safe_*)
  - winRate = wins/n * 100  — only populated when n ≥ 30, else null
  - roi     = sum(roiPercent)/n — only populated when n ≥ 30, else null
  - Returns Map<string, MarketGradedStats>

Function: buildingValue(n, realValue)
  if (realValue !== null && n >= 30) return realValue
  else return `Building (${n}/30)`

Threshold constant: MIN_SAMPLE = 30

The map is computed once in MLBSubscriberView and passed down through
VerdictHeroCard → VerdictPickPanel → VerdictEliteSection → VerdictGameCard
so N ticks up automatically as gradedPicks accumulates real data.


POWER SCORE WEIGHTS
-------------------
Constants (all in MLBSubscriberView.tsx):
  PS_EDGE_W = 3.0   (edge is the dominant signal)
  PS_PROB_W = 0.4   (win probability above 50% adds secondary weight)

Formula:
  powerScore = edge * 3.0 + max(0, winProb - 50) * 0.4

Components intentionally EXCLUDED until data exists:
  - CLV           (only available post-grade, not for today's picks)
  - Historical ROI (building field, no real data yet)
  - Line movement  (not tracked)

BotD eligibility gate (must pass BOTH for BotD or Elite tier):
  BOTD_EDGE = 5   (edge ≥ 5%)
  BOTD_PROB = 70  (calibrated win prob ≥ 70%)

Qualifying display gate (shown at all in subscriber view):
  EDGE_MIN  = 3   (edge ≥ 3%)
  PROB_MIN  = 65  (win prob ≥ 65%)

BotD selection: among all BotD-eligible picks for non-postponed games,
the one with the highest powerScore is promoted; all others are demoted
to "Elite". Exactly one BotD per day.
